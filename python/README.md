# msTrawler (Python)

Pure-Python implementation of the msTrawler proteomics analysis pipeline. Analyzes multiplexed (TMT) mass spectrometry data with signal-aware weighted modeling — no R dependency required.

## Installation

```bash
cd python
pip install -e ".[dev]"
```

**Dependencies:** numpy, pandas, scipy, statsmodels (all from PyPI).

## Quick Start

```python
from mstrawler.file_converter import convert_pd_file
from mstrawler.pipeline import ms_trawl
import pandas as pd

# Step 1: Convert Proteome Discoverer output
df = convert_pd_file("pd_export_PSMs.txt", "pd_export_Proteins.txt")

# Step 2: Load sample and covariate metadata
sample_file = pd.read_csv("sample_file.csv")
covariate_file = pd.read_csv("covariate_file.csv")

# Step 3: Run the full pipeline
ms_trawl(
    df=df,
    sample_file=sample_file,
    covariate_file=covariate_file,
    lod=0.01,
    min_above=4,
    col_adjust=0.5,
    seed=42,  # for reproducible LOD imputation
)
```

Output CSVs are written to the current directory:
- `Simple.csv` — per-sample protein abundance estimates
- `Factor_<Name>_<Level>.csv` — factor contrasts (log2 fold-change, p-value, q-value)
- `Time_<Level>.csv` — time trend estimates and tests
- `ColAdjustmentFactors.csv` — normalization factors applied

## Input Files

### PSM file (from Proteome Discoverer)

Tab-delimited export containing columns:
- `Protein Accessions` or `Master Protein Accessions`
- `Annotated Sequence`
- `Modifications`
- `File ID` (plex identifier)
- `Abundance` columns (one per TMT channel)
- `Average Reporter S/N`

### Sample file (CSV)

Maps TMT channels to experimental conditions:

```csv
SampleID,Bridge,MouseID,Age,Time
F1_X126.Sn,1,Bridge,Bridge,NA
F1_X127n.Sn,0,39,Young,2
F1_X127c.Sn,0,39,Young,4
...
```

### Covariate file (CSV)

Defines the type of each covariate:

```csv
Covariate,Type,Levels,TimeDegree,TimeCategory,Circadian
MouseID,id,0,0,0,0
Age,factor,2,0,1,0
Time,time,0,0,0,1
```

| Type | Meaning |
|------|---------|
| `id` | Subject identifier for random effects grouping |
| `factor` | Categorical variable (e.g., Young vs Old) |
| `continuous` | Numeric covariate (centered automatically) |
| `time` | Time variable for longitudinal modeling |

## API Reference

### `convert_pd_file(pd_psm_file, pd_protein_file="", delimiter="\t")`

Converts Proteome Discoverer exports to an msTrawler DataFrame.

### `ms_trawl(df, sample_file, covariate_file, ...)`

Full analysis pipeline. Key parameters:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `scale_sn` | 1.0 | SNR scaling factor for weights |
| `lod` | 0.01 | Limit of detection (< 1 = proportion, >= 1 = SNR) |
| `impute_penalty` | 1.0 | Penalty for imputed values |
| `min_above` | 4 | Minimum channels above LOD per scan |
| `ssn_filter` | 20.0 | Minimum summed SNR per scan |
| `outlier_cutoff` | 3.0 | Standardized residual threshold for outliers |
| `col_adjust` | 0.5 | SD percentile for normalization protein selection |
| `seed` | None | Random seed for reproducible LOD imputation |

Each pipeline step is also available as a standalone function:

## Pipeline Steps

```
Raw PSM data
  │
  ▼
1. convert_pd_file()     Parse Proteome Discoverer exports
  │
  ▼
2. tmt_lod()             Impute below-LOD values (log-normal, SNR-penalized)
  │
  ▼
3. geo_norm()            Geometric mean normalization across channels/plexes
  │
  ▼
4. find_outliers()       Consensus outlier detection (dual WLS models)
  │
  ▼
5. setup_covariates()    Parse factors, time terms, circadian, build formula
  │
  ▼
6. restructure()         Melt wide→long, merge covariates, compute weights
  │
  ▼
7. adapt_model()         Per-protein weighted model fitting + hypothesis tests
  │
  ▼
8. write_results()       FDR correction (Benjamini-Hochberg) + CSV output
```

## Python vs R: What's Different

### Statistical model layer

| Aspect | R version | Python version |
|--------|-----------|----------------|
| **Linear models** | `lm()` with weights | `statsmodels.WLS` with weights |
| **Mixed-effects models** | `lme4::lmer` (nested random intercepts + weights) | Not available (WLS fallback) |
| **Hypothesis tests** | Kenward-Roger via `pbkrtest` | Wald F-test via `statsmodels` |
| **FDR correction** | `p.adjust(method="fdr")` | `statsmodels.multipletests(method="fdr_bh")` |

### What this means in practice

**The preprocessing is identical** — LOD imputation, normalization, and outlier detection produce the same results as R (verified by E2E tests).

**The model fitting differs** in two ways:

1. **No mixed-effects models.** The R version uses `lme4::lmer` to fit nested random intercepts (subject + sample), which properly separates biological from technical variance. The Python version uses weighted least squares (WLS), which treats all observations as independent. For proteins with many scans per sample, WLS may underestimate standard errors.

2. **Wald tests instead of Kenward-Roger.** The R version uses the KR approximation to compute p-values, which inflates them to account for small-sample uncertainty in variance components. The Python version uses standard Wald F-tests, which are asymptotically equivalent but less conservative for small samples (< 5 subjects per group).

### When does this matter?

| Scenario | Impact |
|----------|--------|
| Large experiments (> 10 subjects per group) | **Minimal.** WLS and Wald converge to lmer+KR as sample size grows |
| Small experiments (3-5 subjects per group) | **Moderate.** P-values may be slightly smaller (less conservative). Mixed effects would better handle within-subject correlation |
| Experiments with few scans per protein | **Minimal.** With 1-2 scans, mixed effects can't be estimated anyway — both versions fall back to WLS/lm |
| Experiments with many scans per protein | **Moderate.** Mixed effects properly down-weight within-sample correlation; WLS may over-count these as independent observations |

### Recommendation

| Use case | Recommendation |
|----------|----------------|
| Integration into a Python service or pipeline | **Python version** |
| Publication requiring mixed-effects models | **R version** (or add Bambi/PyMC Bayesian models to Python — see roadmap) |
| Exploratory analysis and rapid iteration | **Python version** |
| Small-sample experiments where every p-value matters | **R version** |
| Large-scale experiments (production throughput) | **Python version** (easier parallelization, no R runtime) |

## Roadmap: Closing the Gap

The Python version can be enhanced to match or exceed R's statistical capabilities:

1. **Bayesian mixed models via Bambi/PyMC** — no R dependency, better than KR for small samples, native uncertainty quantification. This would replace `adapt_model()` while keeping the rest of the pipeline unchanged.

2. **`statsmodels.MixedLM` with weights** — once [statsmodels issue #9342](https://github.com/statsmodels/statsmodels/issues/9342) is resolved, the existing `MixedLM` class will support observation-level weights, enabling a direct lmer equivalent.

3. **Bootstrap p-values** — resample-based inference as an alternative to both KR and Wald tests. Computationally heavier but assumption-free.
